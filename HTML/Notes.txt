The Developer's Toolkit:
Text Editors: Notepad: Files ending in .txt can be renamed to .html to run, but professional editors are better for efficiency.

UltraEdit: A paid, professional editor for advanced developers and freelancers. It includes code highlighting, templates, CSS color swapping, and SSH/FTP/SFTP integration. It offers a 30-day trial for personal projects.

Visual Studio Code (VS Code): The free, Microsoft-developed editor used in this course. It is likened to a notebook where code is written. 

Installing VS Code: Search "Download VS Code" or visit code.visualstudio.com/download.
Available for Windows (7, 8, 10, 11), macOS, and Linux.
Most systems are 64-bit; select the appropriate version for your machine.
Installation Note: Check all boxes during installation to ensure full functionality.


Web Development Fundamentals:
Front-end Components: HTML (HyperText Markup Language): The structure and layout (like the rooms and blueprint of a house).

CSS (Cascading Style Sheets): The style (colors, button shapes, sizes, carpets, and wall paint).

JavaScript (JS): The functionality and logic (the electrical switches that turn on fans or handle download buttons). 

Level 1: The Basics of HTML:
Exploring Existing Code: View Page Source: Right-click a website (e.g., Google) to see the raw HTML code behind the design.

Inspect Element: A cleaner view showing the "boxes" and "levels" of the layout. It highlights exactly how much space an element (like an image or button) occupies. 
Responsiveness: Websites must adapt to different screen sizes (iPhone, iPad, Pixel). Using the Inspect tool allows you to simulate these various device views and aspect ratios.

First File Setup: Create a folder (e.g., "HTML Tutorial").
index.html: The default home page filename that servers look for automatically. 
VS Code Productivity Tools: Emmet: A built-in toolkit in VS Code for generating code shortcuts.

Boilerplate: The essential starting code generated by typing ! and hitting enter.

Live Server: An extension that opens the HTML page in a browser and refreshes it automatically whenever you save changes.

HTML Tags and Elements: Tag: A container for content (like jars in a kitchen). It consists of an opening tag (e.g., <p>) and a closing tag (e.g., </p> which includes a slash).

Element: The combination of the opening tag, the internal content, and the closing tag. 
Boilerplate Anatomy: <!DOCTYPE html>: Declares the document as HTML5.

<html>: The root element containing the entire document.

<head>: Contains meta-information (data about data) that is not visible on the page.

Meta Viewport: (e.g., <meta name="viewport" content="width=device-width, initial-scale=1.0">). This tells the browser the website is responsive. It ensures the site adjusts its size and aspect ratio for different screen widths (mobile vs. laptop) so it doesn't look broken on phones.

Title Tag: Sets the text for the browser tab (e.g., "My First Page"). 
<body>: Contains all the data (text, images, buttons) that will be rendered and displayed to the user. 

Tag Relationships: <html> is the parent; <head> and <body> are its children and are siblings to each other.

Special Tag Types: Empty Tags: Tags like <br> (line break) that do not have a separate closing tag or internal content.

Comments: Written as <!-- comment -->. They are for developers to read and are not parsed or displayed by the browser. 

Key Rules: Case Sensitivity: HTML is not case-sensitive; <p> and <P> are identical to the browser.

MDN Reference: VS Code provides links to Mozilla Developer Network (MDN) for documentation on browser compatibility and examples. 
Project 1 (Part 1): Create a "Portfolio" folder, initialize index.html, and print "Portfolio" on the screen. 


Level 2: HTML Tags in Detail:

Attributes: Used to add properties to tags (e.g., <html lang="en">). They follow the name="value" format using single or double quotes.

Heading Tags: <h1> (most important/largest) to <h6> (least important/smallest). Use these for SEO and importance ranking, not just to change text size.
Paragraph Tag: <p> for blocks of text.

VS Code Tip: Use "Toggle Word Wrap" in the Command Palette to keep long lines of text visible without horizontal scrolling.

Anchor Tag (<a>): Used for links via the href attribute. Absolute Links: Links to external sites (e.g., google.com).

Relative Links: Links to internal files (e.g., hello.html) within your project folder. 
Image Tag (<img>): Adds images to the page. Attributes: src (source/link) and alt (alternate text if the image fails to load).

Resizing: Use height and width attributes. Note: If you set only one, the aspect ratio is maintained; setting both manually can distort the image. 
Text Formatting Tags: Bold: <b>, Italic: <i>, Underline: <u>.
Relative Text Size: <big> and <small>.

Horizontal Ruler: <hr> creates a line to separate sections.

Subscript: <sub> (e.g., the '2' in H₂O).

Superscript: <sup> (e.g., the '2' in 3²). 

Pre Tag (<pre>): Displays text exactly as written in the editor, preserving all spaces and line breaks that the <p> tag usually ignores.

Project 1 (Part 2): Create separate Education, Experience, and Projects pages and link them to the home page. 


Level 3: Layout and Semantic HTML:

Semantic vs. Non-Semantic: Semantic Tags: Tags that describe their meaning (e.g., <header>, <footer>, <main>). They improve SEO ranking, readability, and accessibility for screen readers.

Non-Semantic Tags: Tags without inherent meaning (e.g., <div>, <span>). 
Structural Layout Tags: <header>: For the top menu and site title.
<main>: For the primary content.

<footer>: For the bottom contact and copyright info.

<section>: Divides content into thematic parts (e.g., Education section).

<article>: For independent content like blog posts or "My Story".

<aside>: For content indirectly related to the main topic (e.g., ads). SEO often ignores this. 

Advanced Features: Target Attribute: Adding target="_blank" to an anchor tag opens the link in a new tab.

Clickable Images: Nest an <img> tag inside an <a> tag. 

Display Types: Block Elements: Occupy the full width of the screen (e.g., <div>, <p>, <h1>, <footer>).

In-line Elements: Occupy only the width necessary for their content (e.g., <span>, <a>, <img>, <b>, <sub>, <sup>). 

General Containers: <div>: A block-level container used to group elements for collective styling (like applying a background color to a group).

<span>: An in-line container used for small portions of text or elements. 

Project 1 (Part 3): Improve the portfolio with a semantic layout and clickable social media icons. 


Level 4: Lists, Tables, and Forms (Pro Level):

Lists: Unordered (<ul>): Bulleted lists. Bullet shapes (circle, square, disc) can be changed in CSS.

Ordered (<ol>): Numbered or sequenced lists (1, 2, 3 or A, B, C).

List Item (<li>): Defines each entry. You can nest lists inside list items to create sublists. 

Tables: <table>: The root table container.
<tr>: Defines a table row.
<th>: Defines a header cell (bold/centered by default).
<td>: Defines a data cell.

Layout Helpers: <caption> for the table title; <thead> and <tbody> for semantic grouping of rows.

Colspan: Merges multiple columns into one (similar to merging cells in Excel). 

Forms: Used to collect user data (Logins, Sign-ups). <form>: Uses the action attribute to define the destination of the data (e.g., a .php file).

Input Types: type="text": Standard text input. Use placeholder for background guide text.

type="password": Masks input with dots for security.

type="radio": Selection from a group. All options must share the same name attribute so only one can be selected.

type="checkbox": Allows multiple selections (e.g., favorite subjects).

type="submit": Creates a button to send the form data. 

Labels: Essential for accessibility. Using the for attribute (matching an input's id) makes the label text clickable, which is helpful for users with small screens or poor eyesight.

ID vs. Class: An ID is a unique identifier for a single element. A Class is for grouping multiple elements for shared styling.

Text Area: <textarea> for multi-line feedback. Sized using rows and cols attributes.

Select Tag: Creates a dropdown menu using internal <option> tags. 

Advanced Media and Embedding: iFrame: Embeds another website or YouTube video. Note: Professional sites like Google or MDN often block this for security, but sites like Wikipedia allow it.

Video Tag: Adds video files (.mp4, .mov). Attributes: src (source), controls (play/pause/volume), loop, autoplay, and muted.

Caution: Avoid unmuted autoplay as it can be annoying and consume user data.
